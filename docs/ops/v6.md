## 배경

my-atlas는 여러 Claude가 각자 feature branch에서 병렬 개발하기 위해 **git worktree**를 사용한다.

```
my-atlas/                          # 메인 (develop)
.claude/worktrees/
  ├── feature-registry/            # worktree 1
  ├── knowledge-base/              # worktree 2
  └── senior/                      # worktree 3
```

각 워크트리에서 `docker compose up`으로 로컬 확인이 필요하지만, 현재 두 가지 문제로 **워크트리에서 바로 실행이 불가능**하다.

---

## 문제 분석

### 문제 1: `.env` 파일 누락

- `.env`는 `.gitignore`에 포함 → git worktree에 복사되지 않음
- `docker-compose.yml`이 `.env`에서 환경 변수(`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, DB 비밀번호 등)를 읽음
- **결과:** 워크트리에서 `docker compose up` 시 API 키 없이 기동되어 기능 동작 불가

### 문제 2: Docker 볼륨 분리

- `docker-compose.yml`의 `pgdata` 볼륨은 이름 미지정 (anonymous named volume)
- Docker Compose는 **프로젝트 디렉토리명**을 접두사로 사용 → 워크트리마다 다른 볼륨 생성
  - 메인: `my-atlas_pgdata`
  - 워크트리: `feature-registry_pgdata` (빈 DB)
- **결과:** 워크트리에서 기동 시 knowledge_base 등 실 데이터가 없는 빈 DB로 시작

### 문제 3: 포트 충돌

- 메인과 워크트리가 동일 포트(5432, 8080, 5173)를 사용
- 메인에서 이미 `docker compose up` 중이면 워크트리에서 기동 불가
- **결과:** 메인 서비스를 내려야만 워크트리 확인 가능

---

## 해결 계획

### 핵심 아이디어: `docker-compose.override.yml` 자동 생성

Docker Compose는 `docker-compose.override.yml`이 같은 디렉토리에 존재하면 **자동으로 머지**한다.
별도 `-f` 플래그 불필요 → Claude들이 평소처럼 `docker compose up -d`만 실행하면 된다.

`setup-worktree.sh`가 워크트리 이름을 기반으로 고유 포트를 할당한 override 파일을 자동 생성한다.

### 포트 할당 표

| 환경 | DB | Backend | Frontend | VITE_API_BASE_URL |
|------|-----|---------|----------|-------------------|
| 메인 (develop) | 5432 | 8080 | 5173 | http://localhost:8080 |
| worktree #1 | 5433 | 8081 | 5174 | http://localhost:8081 |
| worktree #2 | 5434 | 8082 | 5175 | http://localhost:8082 |
| worktree #3 | 5435 | 8083 | 5176 | http://localhost:8083 |

워크트리 이름 순서(알파벳)로 슬롯 1~3을 배정한다.

## 최종 요약

### 변경된 파일

| 파일 | 변경 내용 |
|------|-----------|
| `docker-compose.yml` | `pgdata` 볼륨에 `name: my-atlas_pgdata` 추가 |
| `scripts/setup-worktree.sh` | 신규 — .env 심볼릭 링크 + override.yml 자동 생성 |
| `.gitignore` | `docker-compose.override.yml` 추가 |

### 생성된 파일 (각 워크트리)

| 워크트리 | .env | override.yml | DB | Backend | Frontend |
|----------|------|--------------|----|---------|----------|
| feature-registry | 심볼릭 링크 | 생성 | 5433 | 8081 | 5174 |
| knowledge-base | 심볼릭 링크 | 생성 | 5434 | 8082 | 5175 |
| my-senior | 심볼릭 링크 | 생성 | 5435 | 8083 | 5176 |

### 사용법

```bash
# 새 워크트리 생성 후
bash scripts/setup-worktree.sh .claude/worktrees/<이름>

# 워크트리에서 실행 (평소와 동일)
cd .claude/worktrees/<이름>
docker compose up -d
```

Claude들은 기존과 동일하게 `docker compose up -d`만 실행하면 된다. override 파일이 자동으로 머지되어 포트 충돌 없이 동작한다.
