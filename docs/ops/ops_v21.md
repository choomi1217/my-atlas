# Ops v21 — 기능 문서 QA 시니어 관점 보강

## 변경 목적

my-atlas의 5개 기능 메인 명세서 중 **Platform**이 구현 현황 나열 중심으로만 작성되어 있어, QA 시니어가 "이 기능이 왜 중요한지"를 파악하기 어려웠다. QA 실무자의 관점에서 각 기능의 가치와 활용 방법을 문서에 반영했다.

## 변경 내용

### Platform 명세서 (`platform.md`) → v7

1. **개요 재작성**: "횡단 관심사 담당" → QA 자산 보호 필요성 중심 서술
2. **QA 시니어가 이 기능을 중요하게 봐야 하는 이유** 섹션 추가
   - QA 자산 보호 (JWT 인증)
   - 역할 분리로 실수 방지 (ADMIN/USER)
   - 일관된 작업 환경 (사이드바 통합)
3. **실무 활용 시나리오** 섹션 추가
   - 신규 QA 온보딩 (USER 권한 안전 학습)
   - 리드 QA 팀 지식 큐레이션 (ADMIN 권한 관리)
4. **다른 기능과의 연계** 섹션 추가
