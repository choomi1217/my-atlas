---
name: DB 데이터 삭제 금지 (스키마 변경 없는 경우)
description: DB 스키마 변경이 아닌 작업에서 기존 데이터를 지우지 말 것
type: feedback
---

DB 스키마 변경이 없는 작업에서는 기존 데이터를 삭제하지 않는다.

**Why:** 매번 개발 후 DB가 비워지는 현상이 반복되어 사용자가 수동으로 데이터를 다시 넣어야 했음. 테스트 데이터든 운영 데이터든 보존이 기본이다.

**How to apply:**
- `docker compose down -v` 사용 금지 (볼륨 삭제로 전체 데이터 소실)
- E2E 테스트 후 테스트 데이터 cleanup은 테스트가 생성한 데이터만 정리 (기존 데이터 보존)
- Flyway 마이그레이션 추가 시에도 기존 데이터 유지되는 방향으로 작성
- Agent-D 실행 시 `docker compose down` (볼륨 없이)만 사용
